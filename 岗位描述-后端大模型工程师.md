# 岗位描述（整理版）

## 岗位定位

我们正在寻找一位技术扎实、心态开放的资深后端工程师，以大模型工程化落地为核心方向，加入核心技术团队。  
要求熟练掌握 Python/Java，具备独立主导系统架构设计的能力，在快速演进的 AI 技术浪潮中保持敏锐的技术嗅觉与持续学习的动力。  
期望候选人不仅是执行者，更是能主动洞察问题、提出建议、与团队共创方案的伙伴。  
有金融数据行业背景者优先。

---

## 岗位职责

1. 以大模型工程化为核心，负责 LLM 应用的后端架构设计与落地，涵盖模型服务部署、API 封装、上下文管理、RAG 流程构建等关键环节。  
2. 熟练调用主流大语言模型 API（OpenAI / Claude / Qwen / Llama 等），深入理解不同模型在能力边界、上下文窗口、调用成本与适用场景上的差异，结合业务需求合理选型。  
3. 主导后端系统整体架构设计，对技术选型、模块拆分、稳定性与扩展性全面负责，制定并推动团队工程规范落地。  
4. 基于 Python / Java 进行后端服务开发，构建高并发、高可用的 RESTful / GraphQL API，主导接口规范与数据契约制定。  
5. 负责产品后端功能开发与联调，与 PE、前端、产品团队高效对接，主动识别潜在问题并及时提出优化建议。  
6. 主导数据库建模与查询优化，覆盖 PostgreSQL / MySQL / MongoDB 及 Redis 缓存层；推动消息队列在异步与事件驱动场景中的规范应用。  
7. 负责容器化部署（Docker / Kubernetes）与 CI/CD 流水线建设，配合完善可观测性体系（Prometheus / Grafana）。  
8. 主导 Code Review，推动测试体系建设，承担部分技术 mentor 职能，持续提升团队整体工程质量。  

---

## 任职要求

### 大模型工程能力（必备）

- 具备 2 个及以上完整大模型相关项目的后端工程化落地经验，涵盖模型服务部署、API 封装、多轮上下文管理及异常处理。  
- 熟练调用主流 LLM API，深入了解 GPT / Claude / Qwen / Llama 等模型差异（能力边界、上下文窗口、成本结构、适用场景），能独立完成业务场景下的模型选型决策。  
- 熟悉 RAG 架构，有向量数据库（Chroma / Milvus / Qdrant）或 LangChain / LlamaIndex 实际使用经验。  
- 具备对 LLM 输出进行结构化解析、异常兜底与安全过滤的工程实践，了解 Function Calling、Tool Use 等进阶能力的应用场景。  

### 后端工程与架构能力（必备）

- 计算机相关专业本科及以上学历；3 年以上后端开发经验，有独立主导系统设计或架构重构的实际案例，对自身负责系统有清晰的架构规范意识。  
- 熟练掌握 Python/Java，深入理解各自语言特性：Python 侧须掌握 asyncio、类型注解、设计模式及性能调优；Java 侧熟悉 Spring Boot 生态者优先。  
- 熟练使用 FastAPI 或 Django REST Framework，具备完整 API 设计、文档化与版本管理经验。  
- 深入掌握关系型数据库事务隔离、索引优化与分库分表策略；熟悉 Redis 高级用法与 Kafka / RabbitMQ 消息队列实践。  
- 熟练使用 Docker / Kubernetes，了解 CI/CD 与可观测性体系建设；具备 Linux 基础与 Git 工作流经验。  

### 素质与协作心态（必备）

- 具备出色的沟通与表达能力，能在跨团队协作中清晰传递技术方案，善于倾听并接受反馈。  
- 主动型人格：不止于执行，能主动发现问题、提出改进建议，并推动方案落地；对自己负责的模块有高标准的质量要求。  
- 心态开放，对 AI 领域的快速演进保持持续学习的热情，能主动探索并将新技术、新工具引入日常工作。  
- 拥抱变化，能在技术范式快速迭代的环境中保持稳定输出，善用 AI 工具提升个人与团队效率，而非抵触或观望。  

### 金融数据背景（优先）

- 有金融、支付或数据平台相关系统的开发经验，了解对应业务链路与数据模型。  
- 具备数据一致性、数据安全、数据稽核相关建设经验者优先。
